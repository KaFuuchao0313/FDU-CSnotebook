### 深度学习

寻找一个函数

#### 神经网络

**神经元**

一个简单函数，接受input并赋予权重，得出结果。
输入(x) -> 权重(w)*输入+偏差(b) -> 激活函数 -> 输出(y)
数学表达如下：
$$z = \sum \limits _{i} w_ix_i+b$$
其中，w和b均为可学习参数。
Sigmoid激活函数表达如下：
$$y = \sigma (z) = \frac {1} {1+e^{-z}}$$
它可以将任何实数映射到(0,1)区间内，输出可以解释为概率。

**神经网络**

将激活函数扩充为向量。
数学表达如下：
$$ y = \sigma (Wx+b) $$
此时，W为权重矩阵，输入和输出均为向量表示。注意，激活函数将对逐元素进行运用，每一个单层都需要被激活，以模拟非线性过程。
多层神经网络即将上一层的激活后输出投入本层输入。

**完全连接前馈神经网络**

很多隐藏层。

故训练过程即在函数集中找到最优的那个函数。

### CNN
- 局部连接

#### 图像分类任务

**交叉熵损失函数**

数学表达如下：
$$z = -\sum \limits _{i} y_iln(y_i')$$
其中，y为真实概率，y'为预测概率。

**数字化表示**
一张彩色图像可以表示为三维张量：高度 * 宽度 * 通道数。通常而言，彩色图像通道数为3，灰色图像通道数为1。
例如，在RGB彩色图像中：
- 通道0：红色分量
- 通道1：绿色分量
- 通道2：蓝色分量
被分割而成的每一个像素的值代表颜色强度（0~255）。
在灰色图像中，0=黑色，255=白色。

**感受野**

显然判断并不需要全局，有时抓住局部特征是关键。
于是kernel孕育而生。
感受野可以不覆盖全部输入通道，可以大小不同，可以互相重叠，可以不为正方形，具备灵活性。

设置概念：
- stride 步长
- padding 边缘填充

**参数共享**

同一个卷积核在全部位置使用，神经元参数一致。

**卷积计算**

按照stride移动kernel，与原矩阵部分进行点积，逐层进行，得到feature map。
如果有64个不同的卷积核，那么我们将把feature map合并作具有64 channels的特征图像而进一步进行卷积。
于是，特征图尺寸越来越小，通道数越来越多。

**池化层**

同样使用池化核，按照步长对特征图进行降解。

类型：
- 最大池化（取max）
- 平均池化（取min）
- 全局池化（全部归一）
通常而言，我们采用最大池化类型。

于是，完整的CNN构架横空出世：
输入 -> [ 卷积层 -> 激活函数 -> 池化层 ] * n -> 展平 -> 全连接层 -> softmax -> 输出

```python
# 定义2d卷积层
class LeNet(torch.nn.Module):
  def __init__(self):
    super(LeNet, self).__init__()
    # .Sequential()自动forward
    self.conv1 = torch.nn.Sequential(
      # .Conv2d()定义二维卷积层
      # 识别灰色图像时input为1
      torch.nn.Conv2d(1, 6, kernel_size=5),
      torch.nn.ReLU(),
      torch.nn.MaxPool2d(kernel_size=2),
    )
    self.conv2 = torch.nn.Sequential(
      torch.nn.Conv2d(6, 16, kernel_size=5),
      torch.nn.ReLU(),
      torch.nn.MaxPool2d(kernel_size=2),
    )
    self.conv3 = torch.nn.Sequential(
      torch.nn.Conv2d(16, 120, kernel_size=5),
      torch.nn.ReLU(),
    )
    self.fc = torch.nn.Sequential(
      torch.nn.Linear(120,84),
      torch.nn.Linear(84,10),
    )

  def forward(self, x):
    # 自动计算batch数量防止溢出
    batch_size = x.size(0)
    x = self.conv1(x)
    x = self.conv2(x)
    x = self.conv3(x)
    # 自动展平中
    x = x.view(batch_size, -1)
    x = self.fc(x)
    return x

  model.train()

def evaluate_model(model, test_loader):
  model.eval()
  correct, total = 0, 0
  # 禁用梯度
  with torch.no_grad():
    for images, labels in test_loader:
      outputs = model(images)
      _, predicted = torch.max(outputs.data, 1)
      total += labels.size(0)
      # .item()把张量转化成数字
      correct += (predicted == labels).sum().item()
  accuracy = 100 * correct / total
  print(f'Accuracy of the network: {accuracy:.2f}%')
  return accuracy
```

**文件格式**

#### 激活函数

**sidmoid**

$$ f(z)=\cfrac{1}{1+e^{-z}} $$

映射到（0，1）之间。
缺点有：梯度饱和、不以0为中心、计算慢。

**tanh**

$$ f(z)=\cfrac{e^{z}-e^{-z}}{e^{z}+e^{-z}} $$

映射到（-1，1）之间。
在0附近趋近于线性，但同样会因为梯度较小导致梯度消失问题。
一般二分类问题中，隐藏层用tanh激活，输出层用sigmoid激活，可灵活改变调试。

**ReLU**

$$ f(z)=\begin{cases} 0 &\text{if }z<0 \\ z &\text{if }z\ge 0 \end{cases} $$

优点有：在输入为正数时规避梯度消失问题，因为是线性所以计算速度很快。
缺点有：在输入为负数时完全无法激活，同样有中心不为0的问题。

**ELU**

$$ f(z)=\begin{cases} a(e^{z}-1) &\text{if }z<0 \\ z &\text{if }z\ge 0 \end{cases} $$

对ReLU进行了改进，使其在负数时可以被激活。但是梯度消失和指数运算的问题仍待解决。

#### 损失函数

之前我们所用的交叉熵，一般在图像分类中使用。

**LogSoftmax**

给定向量z=[z1,z2,...,zn]，有Softmax函数：
$$ Softmax(z)_i = \cfrac{e^{z_i}}{\sum \limits _{j=1} ^n e^{z_j}} $$
LogSoftmax函数则是在其基础上对其取对数。一般搭建于多分类任务的最后一层。

**L1**

$$ S=\sum \limits _{1} ^n |Y_i-f(x_i)| $$

**L2**

$$ S= \sum \limits _{1} ^n (Y_i-f(x_i))^{2} $$

**smooth L1**

$$ smooth_{L1}(x)=\begin{cases} 0.5x^{2} &\text |x|<1 \\ |x|-0.5 &\text otherwise \end{cases} $$

改善了L1具有折点而不连续的问题，避免导数不变或过大，收敛精度更高。

## RNN和语音识别

### RNN & LSTM

显然，语音识别应为序列数据，具有时间或顺序依赖性。除此之外，还有词性标注、机器翻译以及情感分析。

#### 前置考量

**Slot Filling**

给定一个句子，识别其中关键词的槽位(slot)。

**词向量**

引入1-of-N encoding，若词表内含有五个词，则用一个1*5的向量组进行表示。但是当词汇量很大时，其维度太高，且大部分元素为0，又无法体现词与词之间的关系。
引入哈希词表，把单词拆分，例如把apple拆成app、ppl、ple。每一种字符组合对应向量中的一个维度。

**Embedding**

将离散符号（单词、字符、类别）映射到连续向量空间，通过训练学习语义关系。

训练流程：
随机初始化 -> 训练更新 -> 预训练嵌入

目前前馈网络最大的局限就是它需要记忆才能判断语义。

#### RNN基本原理

**核心思想**

通过隐藏层实现记忆。
输出依赖当前输入和隐藏状态。

$$ h_t=f(W_hh_{t-1}+W_xx_t+b) $$
其中，xt为该时刻输入，ht-1为上一轮记忆。
$$ y_t=g(W_yh_t+c) $$

显然，所有网络共享参数，且隐藏状态下的记忆会在时间步之间传递，按序列处理。
RNN也可以堆叠成深层网络，设置参数num_layers。
同时也存在双向RNN，从上下文判断语义，设置参数bidirectional=True。

#### LSTM结构

鉴于RNN存在梯度消失的问题，LSTM引入了门控机制，控制信息的流动：输入门，遗忘门，输出门。
所以实际上，LSTM有4个输入（输入门信号、遗忘门信号、输出门信号、输入）和1个输出。

**门控机制**

| 门 | 效果0 | 效果1 |
| --- | --- | --- |
| 输入门 | 不写入新信息 | 写入新信息 |
| 遗忘门 | 遗忘之前的记忆 | 保留记忆 |
| 输出门 | 不输出 | 输出 |
因此，LSTM的参数是普通RNN的四倍，有四组权重，参数量 = 4*(d\*h+h\*h+h)（输入维度d -> 隐藏层h -> 输出层1）。

**训练**

在RNN训练过程中，每一个时间步都有一个标签。
损失函数会在每一个时间步计算损失，最后求和。
其中便有BPTT算法，用以反向传播。它会倒带回去一步一步求解不同时间步下的梯度，然后再相加更新（参数共享）。

**梯度问题**

损失曲线极度不稳定。
因为误差表面粗糙，导致梯度变化剧烈：
- 大梯度导致训练不稳定
- 小梯度导致训练缓慢

为此，有以下解决措施：
- 梯度裁剪（e.x.）
- LSTM（基于加法更新记忆而非乘法）
- GRU（简化至两个门的LSTM）

### 语音识别

#### 基本框架

解码器包含两个模型：语言模型，吃文本数据；声学模型，吃语音数据。
其中声学模型需要特征提取和预处理。

#### 预处理

**分帧**

盯帧。
帧长取20-50ms。

**加窗**

使两端变平缓。
步长为帧偏移，取法一般是帧长的一半或者10ms。

**傅里叶变换**

检测某一频率周期在函数中出现的程度。
$$ F(w)=\int _{-\infty} ^{+\infty} f(t)e^{-iwt} dt $$
此时，将时域信号转化为频域信号，方便处理。

**特征提取**

- 时域特征：声音信号的幅度变化
- 频域特征：可见的频率成分
- Fbank特征：保留频率分布的频谱能量提取
- MFCC：转化为倒谱特征

#### 概率模型

$$ W^*=maxP(X|W)P(W) $$
其中，P(X|W)为声学模型，表示在给定词汇序列W的情况下生成信号X的概率；P(W)为语言模型，表示词序列W的先验概率。

**声学模型**

GMM -> HMM-GMM -> 神经网络

#### CTC

检测音频信号长度通常大于目标文本长度，信息不对齐。通常，它会增加空白符并自动学习对齐，原理是合并相同字符并忽略空白。
其损失函数即计算所有可能生成最终目标文本的路径概率之和。
这种前向-后向的算法高效低计算，然后再反向传播更新网络的参数。

**模型定义**

```python
class CTC(torch.nn.Module):
  def __init__(self, input_dim, output_dim, dropout=0.1):
    super(CTC, self).__init__()
    self.lstm = torch.nn.LSTM(input_dim, hidden_dim, num_layers=2, batch_first=True, bidirectional=True, dropout=dropout)
    self.dropout = torch.nn.Dropout(dropout)
    # 使用bidirectional会使维度翻倍
    self.fc = torch.nn.Linear(hidden_dim*2, output_dim)

  def forward(self, x):
    x,_ = self.lstm(x)
    x = self.dropout(x)
    x = self.fc(x)
    return x
```

## Transformer和机器翻译

### Transformer

#### 序列模型

**输入**

- 单个向量：图像分类
- 向量序列：文本、语音、视频

**输出**

可以是单个标量/类别，上类别也可以对单个或者多个上，至于输出任意长度序列，多是机器翻译（Sequence-to-Sequence）。

#### 自注意力机制

**全连接层的缺点**

- 固定输入输出
- 无法覆盖整个序列
- 参数积累

于是，尝试寻找一种可以考量整个序列的上下文并捕捉任意位置的依赖关系的参数高效机制。

**原理**

自注意力机制要求计算序列中每个位置和其他所有位置的相关性，然后聚合信息。


| 概念 | 含义 | 作用 |
| --- | --- | --- |
| Query | 查询 | 当前位置需求 |
| Key | 键 | 被查询者特征 |
| Value | 值 | 真实语义信息 |

**流程**

1. 计算Q：把当前位置ai与需求矩阵Wq相乘
2. 计算K：把所有位置aj与特征矩阵Wk相乘
3. 注意力分数：QK点乘
4. Softmax：以e的指数为个除以总和
5. 计算V：把所有位置aj与真实矩阵Wv相乘
6. 加权求和：V与softmax结果加权求和

就此引入批量计算：Q=WqA（K、V同理）
其中，A=[a1,a2,...,an]T
O=A'V=softmax(QK^T/√dk)
其中，dk是K的梯度，防止梯度消失的保护措施。

**多头自注意力**

设置不同的“头”来关注不同类型的依赖关系。
可以在nn.MoltiheadAttention中定义头数（num_heads）。
最后把不同头的输出结果组合为向量，乘以拼接矩阵以得到答案。

**位置编码**

自注意力中本身没有位置信息，于是引入e^i作为位置向量，i为位置编码。
此时，ai'=ai+ei。

自注意力可以是一种高级的CNN，相较于RNN最大优点在于它的并行性。

**Seq2Seq**

输入一个序列，输出一个序列。
任务具体有：机器翻译、语音识别、文本摘要、系统对话。
其中，使用Encoder-Decoder：
- 编码器（输入序列 -> 固定长度向量）
- 解码器（编码表示 -> 输出序列）

#### Transformer编码器

输入 -> 多头自注意力层 -> 残差连接+LayerNorm -> 前馈网络 -> 残差连接+LayerNorm

**归一化**

$$ LN(x)=\frac{x-a}{t}r+b $$
其中a为所有特征值的均值，t为所有特征的标准差，r和b均是可学习参数。
它的好处在于序列长度可变且不依赖批次大小，每个位置独立归一化，均值为0，方差为1。

**残差连接**

$$ y=x+F(x) $$

方便快速传递梯度，加快学习优化。

#### Transformer解码器

分为三层：
- 掩码自注意力（关注已生成的输出）
- 交叉注意力（关注编码器的输出）
- 前馈网络

**自回归解码器**

逐步生成输出，直到生成<EOS>。
此处忽略编码器的输入。

**掩码自注意力**

为了实现自回归，挡住后面的信息来预测。
基本实现方式是把后面的注意力分数设置成负无穷，然后经过softmax变成归一矩阵。

**输出处理**

通过softmax把输出转换为概率分布，以贪心本质选取概率最高的词输出。
记得增加停止项。

**交叉注意力**

面向对象是非自回归解码器。为了是实现并行输出，我们需要考虑输出长度的预测和多模态问题。
使用CrossAttention，输入来自解码器的Q以及来自编码器的K和V，让解码器看见输入并关注输入的不同部分。
